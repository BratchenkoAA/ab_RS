# Лабораторная работа № 3.1
## [Организация асинхронного взаимодействия микросервисов с помощью брокера сообщений](https://github.com/BratchenkoAA/ab_RS/tree/main/3.1)

В лабораторной работе №3-1 была разработана система асинхронного взаимодействия микросервисов на основе gRPC и RabbitMQ, реализующая три бизнес-задачи варианта 1: валидацию заказов (проверка JSON на наличие обязательных полей и корректность данных), мониторинг сайтов (получение HTTP статус-кодов через GET-запросы) и анализ тональности текста (определение POSITIVE/NEGATIVE/NEUTRAL). С помощью Protocol Buffers был определен контракт gRPC-сервиса, сгенерирован клиентский и серверный код, после чего реализован gRPC-сервер на Python, обрабатывающий три метода. Для организации асинхронной очереди сообщений был развернут RabbitMQ через Docker Compose, созданы Producer для отправки сообщений в очередь и Consumer для их получения и последующего вызова gRPC-сервера. В результате была построена отказоустойчивая и масштабируемая архитектура, где Producer и Consumer не зависят друг от друга напрямую, а сообщения сохраняются в очереди до обработки.
