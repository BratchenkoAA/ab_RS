### 3.3 Теоретический расчет полосы пропускания

Формула расчета:

```
Bandwidth (бит/с) = (ActiveNodes × Fanout × PacketSize × Overhead × 8) / Interval
```

где:
- ActiveNodes = N × (1 - Failures%) = 15 × 0.75 = 11.25
- PacketSize = 1024 байт
- Overhead = 1.2 (20% на заголовки)

## 4. РЕАЛИЗАЦИЯ ПРОТОКОЛОВ

### 4.1 Протокол Gossip (Serf)

**Принцип работы:** Каждый узел периодически просыпается и обменивается информацией со случайными соседями (fanout). Информация распространяется "вирусно".

**Ключевые параметны:**
- `gossip_interval` - интервал между раундами
- `fanout` - количество соседей для обмена

### 4.2 Протокол Heartbeat (Full-mesh)

**Принцип работы:** Каждый узел пингует ВСЕХ остальных узлов каждый такт.

**Особенности:** 
- Самая быстрая сходимость
- Квадратичный рост трафика O(N²)

### 4.3 Протокол Ping (Random)

**Принцип работы:** Каждый узел пингует ОДНОГО случайного соседа каждый такт.

**Особенности:**
- Низкий трафик O(N)
- Медленное обнаружение отказов

   - При интервале 0.1с трафик в [ЗНАЧЕНИЕ] раз выше, чем при 2.0с

3. **Рекомендации для малых сетей (N=15):**

   - Компромиссный вариант: interval = 0.5с

### 7.2 Выводы по сравнительному анализу протоколов

1. **Heartbeat (Full-mesh):**
   - ✅ Самый быстрый (обнаружение за 0.2-0.4с)
   - ❌ Огромный трафик O(N²) - [ЗНАЧЕНИЕ] сообщений
   - Применим только для малых сетей (<50 узлов)

2. **Gossip (Serf):**
   - ✅ Лучший баланс скорости и трафика
   - ✅ Обнаружение за 1-3 секунды
   - ✅ Трафик растет линейно O(N)
   - 🏆 **Лучший выбор для большинства систем**

3. **Ping (Random):**
   - ✅ Самый экономичный по трафику
   - ❌ Очень медленное обнаружение (десятки секунд)
   - Подходит только для систем, где скорость не критична
